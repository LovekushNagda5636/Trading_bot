PROJECT OVERVIEW
I am building a self-learning AI-powered intraday trading bot for Indian markets. It trades Equity, Futures & Options (NFO), Commodities (MCX), and Currency Derivatives (CDS) via Angel One SmartAPI. The bot has 50+ strategies, ML pipeline, regime detection, adaptive learning, and a dashboard. It is currently a paper-trading bot using real market data.

I am going through every single source file in the project and having it enhanced to production quality. The goal is a fully working, low-risk, maximum-accuracy trading system.

KEY PRINCIPLES FOR ALL ENHANCEMENTS
LOW RISK FIRST вЂ” Capital preservation over profit. Even if profit is small, never take excessive risk.
Everything must actually work вЂ” No placeholders, no broken imports, no stub logic. Every function must be complete.
F&O + Commodity support вЂ” Every component must handle Equity, Futures, Options (NIFTY/BANKNIFTY/Stock), and MCX commodities (Gold, Silver, Crude, Natural Gas, etc.) with correct lot sizes, tick sizes, margin, expiry, settlement.
Segment-aware вЂ” Market hours differ (NSE 9:15-15:30, MCX 9:00-23:30, CDS 9:00-17:00). Transaction costs differ. Risk limits differ per segment.
Self-learning вЂ” The bot learns from every trade: strategy weights, SL/target multipliers, score thresholds, best/worst hours, regime-specific behavior. All learning persists to disk.
Thread-safe, production-grade вЂ” Proper error handling, logging, edge cases, concurrent access patterns.
Backward compatible вЂ” Old code that imports from enhanced files must not break. Keep aliases and legacy fields where needed.
Inter-file consistency вЂ” All files must use the same models, event types, config structures. No duplicate definitions that contradict each other.

WHAT THE BOT DOES (FROM README)
Self-learning intraday bot v2.0 вЂ” learns from every trade, gets smarter over time
Markets: 130+ F&O stocks scanned every 5 min, NIFTY/BANKNIFTY options (directional + straddles), stock futures, MCX commodities
Strategies: 8 equity strategies + 4 options strategies + 3 futures strategies, organized into 6 categories (breakout, trend, reversion, price_action, volume, fno)
Self-learning engine:
Adaptive strategy weights learned from outcomes
Dynamic SL based on volatility + learned multiplier
Adaptive targets + trailing stops
Kelly criterion position sizing scaled by confidence
Learned optimal threshold per regime
Avoids historically bad hours
Regime-aware position limits
End-of-day learning cycle with mistake classification
Mistake classification: tight_sl, wrong_direction, late_entry, weak_signal, low_volatility_entry, premature_exit
Risk management: Max 5% daily loss, 2% per-trade Kelly-scaled, trailing stops, partial exits, EOD auto square-off at 15:25
Currently paper trading вЂ” simulates trades, uses real Angel One market data
INTEGRATION GUIDE SUMMARY (Enhanced Risk Management)
The system has a detailed integration guide for risk management with:

Config sections: risk_limits, position_sizing, stop_loss in JSON config

Position sizing: Kelly criterion with regime and confidence inputs

Stop-loss system: Multi-layered:

Initial adaptive stop (ATR + support/resistance + volatility)
Trailing stops with ratchet mechanism
Time-based stops (no-profit timeout 30min, afternoon tightening 14:30, force exit 15:20)
Volatility-adjusted multipliers
Trading engine integration:

risk_manager.calculate_position_size(signal, balance, positions, regime, confidence, atr)
risk_manager.calculate_initial_stop_loss(entry, direction, atr, s/r, volatility)
risk_manager.update_stop_loss(position, current_price, atr, volatility) in monitoring loop
Regime detection with 3-bar hysteresis to avoid whipsawing
